# 代码架构概述

> 文件 `_worker.js` 约 5830+ 行，单文件架构（Workers/Pages 标准模式）

---

## 一、全局模块结构

### 1. 入口层 (`export default { fetch }`)
- 请求路由入口，处理所有入站 HTTP/WebSocket 请求
- URL 解析、环境变量初始化、路由分发

### 2. 协议处理层

| 模块 | 函数 | 职责 |
|------|------|------|
| XHTTP | `处理XHTTP请求` + `读取XHTTP首包` | XHTTP 协议双向流代理，解析 VLESS/Trojan 首包 |
| gRPC | `处理gRPC请求` | gRPC 协议双向流代理，帧解析和组包 |
| WebSocket | `处理WS请求` | WebSocket 隧道代理，含 Shadowsocks 加解密 |
| TCP 转发 | `forwardataTCP` | TCP 数据流转发核心（直连/反代） |
| UDP 转发 | `forwardataudp` | UDP DNS 转发（8.8.4.4:53） |

### 3. 协议解析层（无外部依赖，纯手写）
- **VLESS**: `解析魏烈思请求` + `UUID字节匹配` + `UUID字节缓存`
- **Trojan**: `解析木马请求`（SHA-224 密码验证）
- **Shadowsocks**: `SSAEAD加密/解密`、`SS派生主密钥`、`SS派生会话密钥`

### 4. 传输层
- **TLS客户端**: ~300 行自定义 TLS 1.2/1.3 实现（`buildClientHello`、`parseServerHello`、`generateKeyShare`、`deriveSharedSecret` 等）
- **代理协议连接器**: `socks5Connect`、`httpConnect`、`httpsConnect`、`turnConnect`、`sstpConnect`
- **连接管理**: `创建上行写入队列`（合包/背压控制）、`connectStreams`、`创建下行Grain发送器`

### 5. 配置层
- `读取config_JSON`: 从 KV + 环境变量构建完整配置对象（每次请求调用）
- `反代参数获取`: 解析 URL 参数设置代理模式
- `识别运营商`: 从 `request.cf` 判断运营商（移动/联通/电信）

### 6. 订阅生成层
- `Clash订阅配置文件热补丁`（~200行）
- `Singbox订阅配置文件热补丁`（~280行）
- `Surge订阅配置文件热补丁`（~20行）
- `请求优选API`（优选IP API 调用和结果解析）

### 7. 辅助工具层
- `MD5MD5`: 双层 MD5 哈希（认证 Token）
- `sha224`: 纯 JS SHA-224 实现（Trojan 密码验证）
- `DoH查询`: DNS-over-HTTPS 客户端
- `base64SecretEncode/Decode`: 带密钥的 Base64 编解码
- `getCloudflareUsage`: Cloudflare API 用量查询

## 二、请求处理流程

```
入站请求
    │
    ├── URL 路由匹配 ──→ /version（版本信息）
    │                    /sub（订阅生成）
    │                    /login（登录页面）
    │                    /admin（管理后台）
    │                    /locations（节点位置）
    │                    其他 → 伪装页反代
    │
    ├── Upgrade: websocket ──→ 处理WS请求
    │                              ├── SS模式（enc参数）
    │                              │      ├── 入站解密（AES-GCM尝试所有候选密码）
    │                              │      ├── SOCKS5地址解析
    │                              │      └── TCP转发/直连
    │                              └── 标准模式
    │                                     ├── 早期数据解码（Base64URL）
    │                                     ├── VLESS/Trojan首包解析
    │                                     └── TCP转发/直连
    │
    ├── POST + gRPC特征 ──→ 处理gRPC请求
    │                           ├── gRPC帧解析（5字节头+载荷）
    │                           ├── VLESS/Trojan首包解析
    │                           └── TCP转发/直连
    │
    └── POST + XHTTP特征 ──→ 处理XHTTP请求
                                ├── 读取XHTTP首包（VLESS+Trojan双解析）
                                └── TCP转发/直连
```

## 三、数据流架构

### 上行方向（客户端 → 目标服务器）
```
客户端
  → WS/gRPC/XHTTP 解析
  → 上行写入队列（合包：目标 16KB）
  → TCP Socket 写入器
  → 远端服务器（直连/反代/SOCKS5链式）
```

### 下行方向（目标服务器 → 客户端）
```
远端服务器
  → BYOB/普通 Reader 读取
  → 下行Grain发送器（合包：目标 32KB）
  → WS/gRPC 帧封装
  → 客户端
```

## 四、关键性能参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 上行合包目标字节 | 16 KB | 小包合并发送阈值 |
| 上行队列最大字节 | 16 MB | 上行队列内存上限 |
| 上行队列最大条目 | 4096 | 上行队列条目上限 |
| 下行Grain包字节 | 32 KB | 下行合包目标大小 |
| TCP并发拨号数 | 2 | 并发连接尝试数 |
| WS早期数据最大字节 | 8 KB | WebSocket 早期数据上限 |
